## 一、核心策略与创新点

### 1.1 研究背景

在A股市场中，动量效应难以捕捉。无论是长端涨跌幅还是短端涨跌幅，在A股市场中均呈现显著的**反转效应**。由于高振幅交易日中，投资者过度反应的概率更高，反转效应更强，并且直接导致长端涨跌幅总体呈现反转效应。

### 1.2 长端动量1.0（基础版本）

**核心思想**: 通过振幅切割的方式，剔除高振幅交易日的涨跌幅，从而剔除过度反应的涨跌数据，去伪存真，构建长端动量因子。

**构建步骤**:

| 步骤 | 操作 |
|------|------|
| 步骤1 | 对选定股票，回溯取其最近160个交易日的数据 |
| 步骤2 | 计算股票每日的振幅：`振幅 = 最高价/最低价 - 1` |
| 步骤3 | 选择振幅较低的70%交易日，涨跌幅加总，得到长端动量因子 |

**关键概念**:
- `Ret_low`: 低振幅交易日（振幅排名前70%，即振幅较小的那70%）的涨跌幅加总 --> 呈现**动量效应**
- `Ret_high`: 高振幅交易日（振幅排名后30%，即振幅较大的那30%）的涨跌幅加总 --> 呈现**反转效应**
- 长端动量1.0 = `Ret_low`（仅使用低振幅交易日的涨跌幅加总）

**1.0版本回测结果**（全市场，2013/01/01 - 2022/10/31）:
- RankIC: 4.01%, RankICIR: 1.76, RankIC胜率: 74.36%
- 5分组多头年化收益: 16.41%, 月均换手率: 35.7%
- 多空年化收益: 10.94%, 年化IR: 1.75, 最大回撤: 10.57%, 月度胜率: 68%

### 1.3 长端动量2.0（改进版本）-- 本报告核心

从以下**4个维度**对长端动量1.0因子进行改进：

#### 改进1: 切割指标优化
- **1.0版本**: 振幅 = 最高价/最低价 - 1
- **2.0版本**: 振幅 = (最高价 - 最低价) / 前收盘价
- **原因**: 使用前收盘价归一化，使振幅更具可比性

#### 改进2: 预处理 -- 剔除涨跌停及停牌交易日
- 剔除停牌日数据
- 剔除涨跌停交易日数据
- **原因**: 停牌日振幅为0，应归为低振幅组但当日涨跌幅无意义；涨跌停交易日振幅受涨跌停交易机制限制，无法充分反映交易活跃程度

#### 改进3: 切割对象 -- 使用个股alpha收益替代原始涨跌幅
- **1.0版本**: 对原始日涨跌幅进行切割加总
- **2.0版本**: 对个股alpha收益（股票日收益 - 市场日收益均值）进行切割加总，记为 `AlphaRet_low`
- **原因**: 股票日收益受市场beta影响。在横截面对比时，若股票日收益对应时点一致，则市场beta不影响股票间相对排序；若股票日收益对应时点不一致，则两股票日收益时点重合度越低，市场beta对股票间相对排序影响越大

#### 改进4: 中性化处理
- 对 `AlphaRet_low` 做**20日反转中性**
- **原因**: `AlphaRet_low` 在反转因子上有一定程度正暴露（相关性18.6%），反转因子越强的月份，`AlphaRet_low` 越弱。通过中性化消除与反转因子的正相关性

**长端动量2.0 构建步骤（完整）**:

| 步骤 | 操作 |
|------|------|
| 步骤1 | 对选定股票，回溯取其最近**160个交易日**的数据 |
| 步骤2 | **剔除涨跌停及停牌交易日数据** |
| 步骤3 | 计算股票每日的振幅：`振幅 = (最高价 - 最低价) / 前收盘价` |
| 步骤4 | 选择振幅较低的**70%交易日**，日超额收益（涨跌幅 - 市场涨跌幅均值）加总，记作 `AlphaRet_low` |
| 步骤5 | `AlphaRet_low` 做**20日反转中性**，得到长端动量2.0因子 |

### 1.4 20日反转中性化的具体方法

研报中提到对 `AlphaRet_low` 做20日反转中性。具体实施方式为：

- 计算20日反转因子：`Reverse20d = 过去20个交易日涨跌幅`（即Ret20，负值代表下跌）
- 在截面上，对 `AlphaRet_low` 关于 `Reverse20d` 做**截面回归**，取残差作为中性化后的因子值
- 即：`AlphaRet_low = alpha + beta * Reverse20d + epsilon`，长端动量2.0 = epsilon（残差）

### 2.4 指数增强框架（沪深300为例）

使用约束优化求解方式在合成因子股票池中进行指数增强测试：

**优化目标**: max alpha^T * w

**约束条件**:
- 风格暴露度偏离上下限: s_l <= X_s(w - w_b) <= s_u，其中 s_l = -0.01, s_u = 0.01
- 行业偏离上下限: h_l <= X_h(w - w_b) <= h_u，其中 h_l = -0.01, h_u = 0.01
- 权重偏离上下限: w_l <= w - w_b <= w_u，其中 w_l = -0.01, w_u = 0.01
- 成分股权重下限: b^T * w >= c_l，其中 c_l = 0.8
- 权重之和为1: 1^T * w = 1
- 权重非负: 0 <= w <= 1

**沪深300增强结果**: 增强组合年化收益率 12.4%, 年化超额收益率 6.28%, 年化IR 1.33, 最大回撤 5.28%（2019.02），月度胜率 60.6%

## 三、数据需求清单

### 3.1 本地数据（~/local_data/）

| 数据文件 | 用途 | 关键字段 |
|---------|------|---------|
| `ashare_stock_price.parquet` | 股票日行情（最高价、最低价、前收盘价、收盘价） | stock_code, date, prev_close, high, low, close, open |
| `ashare_stock_trade.parquet` | 股票交易数据（涨跌幅、振幅、市值、换手率） | stock_code, date, change_pct, range_pct, market_value, negotiable_market_value, turnover_rate |
| `ashare_stock_st.parquet` | ST标记（剔除ST股） | stock_code, implement_date, remove_date |
| `ashare_suspend.parquet` | 停牌标记（剔除停牌股/停牌日） | stock_code, date, ifsuspend |
| `ashare_stock_industry.parquet` | 行业分类（行业中性化） | stock_code, first_industry_name, standard_code |
| `ashare_index_components.parquet` | 指数成分股（沪深300/中证500/中证1000） | stock_code, in_date, out_date, index_code |
| `ashare_tradeday.parquet` | 交易日历 | date |
| `ashare_stock_value.parquet` | 估值数据（可选，用于基本面因子对比） | stock_code, date, pe_ttm, pb_lf |

### 3.2 数据字段说明

- `change_pct`: 日涨跌幅（百分比形式，如 1.0922 表示 +1.0922%）
- `range_pct`: 日振幅（百分比形式）-- **注意**: 需要确认该字段定义是否为 (high-low)/prev_close，若不一致则需要自行计算
- `ifsuspend`: 是否停牌（0=未停牌，1=停牌）
- `market_value`: 总市值
- `negotiable_market_value`: 流通市值

### 3.3 涨跌停判断方法

本地数据中没有直接的涨跌停标记字段，需要通过以下方式判断：
- 涨停: 当日收盘价 >= 前收盘价 * (1 + 涨停板比例 - 容差)
  - 主板: 10% (2020年8月24日创业板注册制改革前)
  - 创业板（注册制后）: 20%
  - 科创板: 20%
  - ST股票: 5%
- 跌停: 当日收盘价 <= 前收盘价 * (1 - 跌停板比例 + 容差)
- 简化处理: 可用 `abs(change_pct) >= 9.5` 作为近似涨跌停判断（考虑四舍五入和ST等情况，可设更精细的阈值）

**推荐做法**: 使用 `close / prev_close - 1` 计算实际涨跌幅，判断是否触及涨跌停板。容差建议设为0.1%。

### 任务3: 振幅计算与交易日分类
**目标**: 计算每日振幅并区分高低振幅交易日

**具体步骤**:
1. 计算每日振幅: `amplitude = (high - low) / prev_close`
2. 对每只股票回溯160个交易日
3. 在160个交易日中，**先剔除停牌日和涨跌停日**
4. 在剩余有效交易日中，按振幅从小到大排序
5. 选取振幅最小的70%交易日，标记为低振幅日
6. 剩余30%标记为高振幅日

**注意事项**:
- 剔除停牌日和涨跌停日后，有效交易日数可能不足160天，按实际有效天数的70%来切割
- 振幅百分位切割时使用 `排名 <= 总有效天数 * 0.7` 来确定低振幅日

### 任务4: 长端动量1.0因子计算
**目标**: 实现基础版长端动量因子

**具体步骤**:
1. 对每只股票的每个截面日期：
   - 回溯160个交易日
   - 计算每日振幅: `amplitude = high / low - 1`（注意：1.0版本使用的是这个公式）
   - 选取振幅最小的70%交易日
   - 将这些交易日的日涨跌幅加总
   - 即为长端动量1.0因子值
2. 使用向量化操作替代逐股票循环

**公式**:
```
LongMom1.0_i = SUM(ret_t * I(amplitude_t <= P70(amplitude)))
其中:
  ret_t = close_t / close_{t-1} - 1（日涨跌幅）
  amplitude_t = high_t / low_t - 1（日振幅，1.0版本定义）
  P70 = 回溯160日中振幅的70%分位数
  I() = 指示函数
```

### 任务5: 长端动量2.0因子计算（核心）
**目标**: 实现改进版长端动量因子

**具体步骤**:

**步骤5.1 - 振幅计算（改进公式）**:
```
amplitude_t = (high_t - low_t) / prev_close_t
```

**步骤5.2 - 预处理**:
- 剔除停牌日（ifsuspend == 1 的交易日）
- 剔除涨跌停交易日

**步骤5.3 - 市场日收益均值计算**:
```
market_ret_t = mean(ret_{i,t}) for all stocks i on day t
```
即每个交易日所有股票日收益率的截面均值。

**步骤5.4 - 个股alpha收益**:
```
alpha_ret_{i,t} = ret_{i,t} - market_ret_t
```

**步骤5.5 - AlphaRet_low 计算**:
对每只股票i在截面日T：
1. 回溯T之前的160个交易日
2. 在这160日中剔除停牌日和涨跌停日，得到有效交易日集合
3. 计算有效交易日的振幅（改进公式）
4. 按振幅排序，选取振幅最小的70%交易日
5. 将这些低振幅交易日的alpha收益加总：
```
AlphaRet_low_i = SUM(alpha_ret_{i,t} * I(t in low_amplitude_days))
```

**步骤5.6 - 20日反转中性化**:
1. 计算20日反转因子:
```
Reverse20d_i = close_T / close_{T-20} - 1
```
即截面日期T相对20个交易日前的涨跌幅

2. 在截面上对AlphaRet_low关于Reverse20d做OLS回归，取残差:
```
AlphaRet_low_i = alpha + beta * Reverse20d_i + epsilon_i
LongMom2.0_i = epsilon_i
```

**最终输出**: 每个月末每只股票的长端动量2.0因子值

### 任务6: 因子预处理（标准化、中性化）
**目标**: 对原始因子值进行标准化和中性化处理

**具体步骤**:
1. **缩尾处理（Winsorize）**: 对因子值做3倍标准差缩尾，或MAD（Median Absolute Deviation）方法
2. **市值中性化**: 在截面上，对因子值关于 ln(市值) 做回归取残差
3. **行业中性化**: 在截面上，对因子值关于行业哑变量做回归取残差（中信一级行业分类）
4. **标准化**: Z-Score标准化（均值0，标准差1）

**注意**: 市值和行业中性化可以合并为一次多元回归:
```
factor_i = alpha + beta * ln(market_cap_i) + SUM(gamma_j * Industry_dummy_j) + epsilon_i
中性化后因子 = epsilon_i
```

### 任务7: 行业长端动量因子（可选扩展）
**目标**: 构建行业层面的长端动量因子

**具体步骤**:
1. 获取各行业成分股
2. 将个股长端动量2.0因子值按自由流通市值加权聚合到行业层面:
```
Industry_LongMom_j = SUM(w_{i,j} * LongMom2.0_i) for all stocks i in industry j
其中 w_{i,j} = free_float_market_cap_i / SUM(free_float_market_cap) for industry j
```
3. 得到每个月末每个行业的长端动量因子值

## 五、关键实现注意事项

### 5.1 向量化实现建议

因子计算涉及每只股票回溯160个交易日的滚动计算，建议采用以下方式提高效率：

1. **将数据转为宽表**（pivot）: 行为日期，列为股票代码，值为对应指标
2. **使用rolling窗口**: 对振幅排序和条件筛选，可以通过rolling window + apply实现
3. **批量处理截面**: 在每个月末调仓日，一次性计算所有股票的因子值

### 5.2 月末调仓日确定

使用 `ashare_tradeday.parquet` 获取交易日历，取每月最后一个交易日作为调仓日。

### 5.3 涨跌停判断的板块差异

不同板块涨跌停比例不同：
- 主板（上海、深圳主板）: 10%
- 创业板（300xxx）: 2020年8月24日前为10%，之后为20%
- 科创板（688xxx）: 20%
- ST/\*ST股票: 5%

可通过股票代码前缀判断板块，再根据日期判断适用的涨跌停比例。

### 5.4 数据字段验证

- `ashare_stock_trade.parquet` 中的 `change_pct` 是百分比形式（如1.09表示+1.09%），在计算时需要除以100转为小数
- `range_pct` 同样为百分比形式
- 需要验证 `range_pct` 的定义是否等于 `(high - low) / prev_close * 100`

### 6.2 核心概念总结
- **动量效应**: 过去收益较高的股票未来继续表现良好
- **反转效应**: A股市场中普遍存在的过度反应导致的价格回调
- **振幅切割**: 通过区分高低振幅交易日，分离动量效应（低振幅日）和反转效应（高振幅日）
- **过度反应**: 高振幅交易日中投资者更容易过度反应，导致反转效应更强

## 七、改进建议

### 7.1 因子构建层面
1. **回看天数敏感性**: 研报固定使用160日，可尝试120/200/240日等不同窗口，检验因子稳健性
2. **低振幅比例敏感性**: 研报使用70%，可尝试60%/80%等不同比例
3. **市场收益均值计算方式**: 可对比等权均值 vs 市值加权均值的差异
4. **反转中性化天数**: 除20日外，可尝试10/30/60日反转因子中性化
5. **振幅定义扩展**: 可尝试使用True Range（考虑跳空缺口）替代简单振幅

### 7.2 风险与局限
1. **样本内过拟合风险**: 回看天数160日和70%比例的选择可能存在过拟合
2. **涨跌停制度变化**: 创业板注册制改革后涨跌停比例从10%变为20%，可能影响因子效果
3. **市场环境依赖**: 动量因子在不同市场环境（牛市/熊市/震荡市）中表现差异大
4. **换手率**: 虽然相比传统量价因子已降低，但34%的月均换手率仍不算低
5. **流动性约束**: 全市场回测中小票占比较大，实际投资中可能面临流动性问题

### 7.3 可扩展方向
1. **机器学习增强**: 用机器学习方法替代简单的70%阈值切割，动态确定最优切割比例
2. **多因子组合**: 与基本面因子（估值、盈利、成长）的组合优化
3. **行业轮动应用**: 利用行业长端动量因子构建行业轮动策略
4. **指数增强实操**: 在沪深300/中证500/中证1000上分别进行指数增强测试
5. **样本外检验**: 将回测数据延长至2022年以后，检验因子的样本外表现
