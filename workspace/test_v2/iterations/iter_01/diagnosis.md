### 家族一 · 盈亏比（13 项）→ 【可修实现口径】，本轮锁定

**归因假设排序：**
1. **[首选，高置信] timing_metrics 盈亏比用「持仓日均值比 mean/mean」，而研报口径为「总额比 sum/sum」。** — 同库铁证坐实。
2. [已排除] 策略信号实现错误 → 排除：核心五指标（区间/年化/回撤/波动/夏普）均同量级，且**无策略逻辑的 T 买入持有基准盈亏比同样偏低**（R1-T 1.013 vs 1.11 -8.77%、R7-T 1.014 vs 1.11 -8.67%），偏差与策略实现无关。
3. [已排除] 研报口径不可推断（v1 旧系统结论）→ 本轮通过 R3 铁证推翻，见下。

**证据链（独立复算）：**
- **同库口径打架（铁证）**：`src/test_v2/strategy.py:245-253` 的 `_odds`（R3 表3 赔率/盈亏比）用**总额比** `sum(f>0)/|sum(f<0)|`，R3 复现值逐格对齐研报（`comparison.json` R3_abs/1%/5% 三档 rel_dev 0.0025~0.0052，全 pass）。strategy.py:219-221 注释明写「经 spec 表格逐格反推为 sum 总额比、非均值比」——**coder 在 R3 已坐实研报盈亏比口径 = 总额比**。

### 家族二 · 年化夏普（9 项）→ 【数据源口径差异，不可修】

- **归因**：国债期货收益序列数据源口径差异（本地 `pct_of_close_price` 同合约日收益 vs 研报 Wind 主力连续合约换月复权），系统性使日收益均值偏高约 +5%。
- **证据**：无策略逻辑的 **T 买入持有基准年化 +5.07% / 夏普 +5.12% 同步偏高**（verify m1 验证点4），而 T 波动率仅 +0.16%（对齐）——日收益离散度正确、仅均值整体偏高，指向收益序列本身而非年化天数口径。v1 旧系统 T 基准年化亦 +5.42%（`output/test/verify_report.md:45,77`），独立复现同一底座偏差，v1 归因「本地主力合约拼接口径与报告 Wind 口径接近但不完全相同」。关联 AS6（换月未特殊处理）。
- **为何 R8/R9/R11 超差而 R1 长短端/R7 主列未超差**：同一 +5% 底座偏高落在 5% 容差边缘——R1 长短端年化 +0.49%、R7 多空 +4.30%（勉强过），R8/R9/R11 年化 +5.3%~+6.3%（刚越线）。本质是同一数据源漂移，非策略叠加引入。
- **不可修**：本地无 Wind 连续合约序列，无法复现研报数据源口径；强改换月拼接会破坏已对齐的波动率/回撤/递增结构，风险高收益负。属数据源不可得的定义差异。

## 三、本轮锁定修改点（1 个 · 最高性价比家族）

> 遵守规则8「每轮最多锁定 1-2 个修改点」。本轮仅锁定 1 个：家族一为唯一「高置信 + 明确可修 + 大批量（13 项）+ 单点改动」的家族；家族二/五不可修（数据源/定义差异），家族四属验证容差层（非 coder src），家族三单腿收敛不确定。首轮聚焦单一高置信修改，便于下轮隔离判断残差。

### 修改点 M1：盈亏比统计口径 由「持仓日均值比」改「持仓日总额比」

**允许 coder 改动的文件范围（越界即违规）：**
- `common/timing_backtest.py` —— **仅** `timing_metrics` 函数内盈亏比计算块（现 150-156 行）。
- 严禁改动：win_rate / sharpe / annual_return / max_drawdown / trade_count 等任何其他指标；严禁改动 `src/test_v2/strategy.py` 的 `_odds`（R3 已对齐研报，是本次修改的口径基准，保持不动）；严禁改动 F1-F8 任何信号逻辑。

**具体修改指令：**
将 `common/timing_backtest.py:150-156` 的均值比：
```python
avg_gain = active_ret[active_ret > 0].mean()
avg_loss = active_ret[active_ret < 0].mean()
profit_loss_ratio = (
    float(avg_gain / abs(avg_loss))
    if pd.notna(avg_loss) and avg_loss < 0 and pd.notna(avg_gain)
    else 0.0
)
```
改为总额比（与 strategy.py `_odds` 的 sum 口径统一）：
```python
sum_gain = active_ret[active_ret > 0].sum()
sum_loss = active_ret[active_ret < 0].sum()   # 负值
profit_loss_ratio = (
    float(sum_gain / abs(sum_loss))
    if sum_loss < 0 and sum_gain > 0
    else 0.0
)
```
- 语义：盈亏比 = 持仓日总盈利额 / 持仓日总亏损额（绝对值），即 `Σ(收益>0) / |Σ(收益<0)|`。
- 同步更新该块 docstring/注释，注明口径与 R3 `_odds` 统一为 sum 总额比。
- 消费侧（`main.py` / `reversal.py` / `combo_ls.py` / `robustness.py` / `combo_composite.py` / `combo_final.py`）均只读 `m["profit_loss_ratio"]`，单点修改全表自动跟随，无需逐处改。

**预期指标变化方向与量级：**
- 方向：全表盈亏比**升高**（均值比 → 总额比，乘以 n_gain/n_loss>1 因子）。
- 量级（逐列预测见家族一证据表）：R1 长短端 1.107→~1.21、R7 多空 1.054→~1.29、R8 多空 1.085→~1.32、R9 多空 1.103→~1.35、R11 多空/R13 1.116→~1.43；仅做多 4 列 →~1.43~1.52。
- 预期收敛：家族一 13 项中 **8-12 项** 收敛入 ±5%（R1 三主列 + R7/R8/R9/R11 多空 + R13 稳收敛；仅做多 4 列大概率收敛）；**R1-T 1 项临界**（预测 ~1.05 vs 1.11，叠加家族二数据源偏差可能仍略超，非本修改点能独立救回）。
- 副作用检查：R3（strategy.py `_odds`，本已 pass）不受影响（独立实现，不走 timing_metrics）；timing_metrics 持仓日样本充足（约 1500+ 日），sum 口径无小样本极端值风险。

**验证方式**：coder 冒烟重跑后，verifier 重生成 `comparison.json`，观察盈亏比家族 pass 数由 0/13 的变化，以及 R3 是否仍全 pass（回归检查）。
