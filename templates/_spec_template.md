# 复现规格书 spec.md 骨架

> `quant-extractor` 生成 `workspace/{report_id}/spec/spec.md` 时遵循本骨架。
> spec.md 回答「研报里有什么必须被复现」；plan.md 只回答「按什么顺序做」——本文件不写实现顺序、不写 milestone 拆分。
> 所有下游文件（coverage_matrix.md / plan.md / assumptions.md / 代码注释 / 审计意见）通过要素 ID 引用 spec.md；**ID 一经分配永不复用、永不删除**（即便后续判定该要素 infeasible，也是改状态，不是撤销编号）。

---

## 一、要素 ID 前缀体系

| 前缀 | 类别 | 编号规则 | 示例 |
|------|------|---------|------|
| D | 数据要求 | D1..Dn | D3 中信一级行业分类 |
| F | 因子/信号/策略定义（含公式） | F1..Fn；子变体用小数位 F3.1 | F2 趋势动量 MA_60 |
| B | 回测设置（区间/股票池/费率/调仓/中性化/分组） | B1..Bn | B4 月末调仓 |
| R | 研报核心数值结果基准 | R1..Rn，通常对应研报一张结果表；行级可用 R1.3 | R1 表3 动量因子测试结果 |
| SA | 敏感性/稳健性/子样本分析 | SA1..SAn | SA1 参数 L 敏感性 |
| FIG / TBL | 研报图/表登记（全集） | **用研报原生编号**，不重新编号：研报「图12」登记为 FIG12 | FIG12 |
| EX | 研报以「图表N」统一编号（不区分图/表）时，替代 TBL/FIG 使用 | EX1..EXn | EX25 |

D/F/B/R/SA 五类写成正文 `### [前缀编号]` 标题块（见「三、要素条目固定格式」）；FIG/TBL/EX 三类写成「六、图表登记清单」的表格行，**不是** `### [ID]` 标题块（两者门禁解析方式不同，见下）。

## 二、frontmatter schema（spec.md 文件最顶部，YAML）

```yaml
---
report_name: <snake_case，与 workspace 目录名一致>
title: <研报标题>
institution: <机构>
report_date: <研报发布日 YYYY-MM-DD>
authors: <作者，多人用、分隔的单个字符串>
market: <A股 | 港股 | 美股 | 多资产>
pdf_pages: <PDF 物理总页数，应与 state.json 的 pdf_pages 一致>
exhibit_declared: {fig_max: <全文出现过的最大图号>, tbl_max: <全文出现过的最大表号>}
element_counts: {D: <n>, F: <n>, B: <n>, R: <n>, SA: <n>, FIG_registered: <n>, TBL_registered: <n>}
type_hint: <factor | timing | allocation | fixed_income | ml>   # extractor 初判，plan 阶段由 planner 定稿
tags_hint: []                                                    # 附加维度初判，如 [ml]；多数研报为空列表，允许为空但键必须存在
extraction_date: <本次提取日期 YYYY-MM-DD>
---
```

**frontmatter 必填字段**（`tools/check_gates.py` G-EX-3 逐一核验非空，`tags_hint` 例外——允许空列表但键必须存在）：
`report_name / title / institution / report_date / authors / market / pdf_pages / exhibit_declared / element_counts / type_hint / tags_hint`。

**门禁锚点（三方一致，G-EX-4）**：
`element_counts` 中 D+F+B+R+SA 五项数值之和 == 正文用正则 `^### \[(D|F|B|R|SA)\d+(?:\.\d+)?\]` 统计出的标题块数 == `coverage_matrix.md` 数据行数。三者任一处漏登、多登、编号错位都会导致该门禁 FAIL——**每新增/修改一条要素，三处必须同步**。

## 三、要素条目固定格式（D/F/B/R/SA 通用骨架）

机器可校验的最小结构：`### [前缀编号] 标题` 一行 + 若干 `- 字段: 值` 行，其中 **`- 页码:` 与 `- 原文:` 两行必填**（`tools/check_gates.py` G-EX-5 逐条要素核验，缺一即 FAIL）。标题行必须顶格、`###` 与 `[前缀编号]` 之间保留一个空格，中括号内不得插空格，门禁正则 `^### \[(D|F|B|R|SA)\d+(?:\.\d+)?\]` 按此匹配统计要素数。

### F 类示例（因子/信号/策略定义）

```markdown
### [F2] 趋势动量因子 MA_60
- 页码: p9-p10
- 原文: > "MA_{j,t,L} = (P_{j,d-L+1} + ... + P_{j,d}) / L……"
- 公式（转写）: MA_bar = mean(close, L=60) / close_d
- 参数: L=60；取值日=月末最后交易日
- 依赖数据: D1 / 关联基准: R2.2 / 关联歧义: A3（如有）
```

> `原文` 行书写要求：≤80 字关键句，数值/单位/正负号原样照抄，禁止转述、禁止在值后追加说明性文字（示例块本身即可直接复制的干净写法）。

### D 类示例（数据要求）——**每条须对照 `templates/data_catalog.md` 标注数据状态**

```markdown
### [D1] 中信一级行业分类（中性化用）
- 页码: p5
- 原文: > "……按中信一级行业进行行业中性化处理……"
- 数据状态: available
- 说明: ashare_stock_industry.parquet, standard_code=37
- 关联要素: F2 / 关联歧义: （如有）
```

> `数据状态` ∈ `{available, derive, missing}`，对照 `data_catalog.md` 判定。

### B 类示例（回测设置）

```markdown
### [B4] 月末调仓
- 页码: p11
- 原文: > "……每月末最后一个交易日调仓……"
- 参数值: 调仓频率=月度；取值日=月末最后交易日；单边费率=0.15%
- 关联要素: F2, B1 / 关联歧义: （如有）
```

### R 类示例（核心数值结果基准）——**只抄录，禁止推算填补**

```markdown
### [R1] 表3 动量因子测试结果
- 页码: p12
- 原文: > "RankIC均值0.052，ICIR 0.68，多空组合年化收益18.3%，夏普1.42……"
- 数值表（与研报表格逐格一致，原样含正负号/百分号；研报未给的格子写 n/a，禁止推算填补）:

  | 指标 | 数值 |
  | --- | --- |
  | RankIC均值 | 0.052 |
  | ICIR | 0.68 |
  | 多空年化收益 | 18.3% |
  | 多空夏普 | 1.42 |
  | 月均换手率 | n/a |

- 关联要素: F2, B4
```

> **R 类铁律**：研报没给的数字写 `n/a`，不得用公式反推、不得用别处数字类比填补——R 类是验证基准的唯一真相源，任何推算值混入都会污染 verify 阶段的容差判定。

### SA 类示例（敏感性/稳健性/子样本分析）

```markdown
### [SA1] 参数 L 敏感性
- 页码: p15
- 原文: > "分别测试L=20、60、120，多空年化收益均值……"
- 测试设计: L ∈ {20, 60, 120} 分别测算，其余参数不变
- 结论: > "……结果对 L 不敏感，均显著为正……"
- 关联要素: F2
```

## 五、图表登记清单格式契约（机器门禁逐字对齐，务必照抄）

第六节「图表登记清单」是**登记全集、复现子集**——研报里出现过的每一张图/表都必须登记一行，但允许标注不复现（`复现意图: skip`）并给理由；**禁止"没登记所以不算漏"**。

**格式规则**（`tools/check_gates.py` G-EX-8 机器核验，逐字对齐，不得走样）：

1. H2 标题必须包含「图表登记」四字（如 `## 六、图表登记清单（FIG/TBL 全集）`）。
2. 登记内容是一张 markdown 表格（表头 + 分隔行 + 数据行），**不是** `### [ID]` 标题块。
3. 数据行首列必须是 `| FIG12 |`、`| TBL3 |` 或 `| EX25 |` 这种「竖线 + 前缀 + 纯数字 + 竖线」形式（门禁正则 `^\|\s*(FIG|TBL|EX)(\d+)\s*\|`）——前缀与数字之间不得有空格、不得加多余符号。
4. **FIG 行数 + EX 行数之和 必须等于 frontmatter 的 `element_counts.FIG_registered`**；**TBL 行数必须等于 `element_counts.TBL_registered`**（EX 前缀计入 FIG 桶，用于研报以「图表N」统一编号、不区分图表类型的情形）。
5. 登记的最大编号不得超过 frontmatter 声明的 `exhibit_declared.fig_max` / `exhibit_declared.tbl_max`（超出即门禁 FAIL——防止漏填 exhibit_declared 或编号抄错）。
6. 编号连续性（是否有编号被跳过未登记）不是本门禁的判定范围，属于 spec_audit 阶段审计 agent 的 C1 职责；本门禁只做计数一致与"不超过声明上限"两项弱校验。

推荐表格样式（列名可读即可，门禁不校验列名，只校验首列格式）：

```markdown
## 六、图表登记清单（FIG/TBL 全集）

| ID | 标题 | 页码 | 摘要 | 复现意图 | 理由/关联要素 |
| --- | --- | --- | --- | --- | --- |
| FIG1 | 因子构造流程图 | p9 | 示意流程，无数值 | reference_only | 示意图，不含可验证数值 |
| FIG2 | RankIC月度时序 | p12 | 月度RankIC柱状图 | reproduce | 关联 R1 |
| TBL1 | 表3 动量因子测试结果 | p12 | 见上 R1 | reproduce | 关联 R1 |
| TBL2 | 附录同业对比表 | p20 | 与本报告方法无关的同业数据 | skip | out_of_scope: 非本报告方法产出 |
```

`复现意图` 取值：`reproduce`（需要复现出对应图表）/ `reference_only`（仅作背景参考，不要求复现）/ `skip`（不复现，理由列必填，如 `out_of_scope` / `data_missing` / `cost_prohibitive`）。

## 六、正文写作规则汇总（供 extractor 自查）

1. **`### [前缀编号]` 标题行正则可统计**：这是门禁计数的唯一依据，标题格式必须逐字对齐「三、要素条目固定格式」，不得自创写法（如 `### F2.` 或 `### [F2]:` 都不匹配）。
2. **R 类只抄录禁推算**：研报没给的格子写 `n/a`，不得用公式反推、不得跨表挪用类比值。
3. **图表用研报原生编号全集登记**：不要为图表重新编号；每张图/表都要有登记意图（reproduce/reference_only/skip），skip 必须给理由。
4. **原文摘录 ≤80 字、数值/正负号/单位原样**：`- 原文:` 行是审计 C6（幻觉引用抽查）与 codex spec_audit 页码真实性抽查的核验对象，转述/意译会被判定为幻觉引用（critical）。
5. **不确定处进「歧义清单」而非脑补**：spec.md 只陈述研报说了什么，任何"研报没写清楚但我判断应该是……"的内容一律写入 `workspace/{report_id}/spec/ambiguities.md`（见 `templates/audit/ambiguities.md`），不得直接写进 spec.md 正文冒充研报原文。
6. **ID 只增不删、永不复用**：审计阶段发现遗漏要素时，追加新编号（如 F13），不得插入 F2.5 打乱既有引用；不得因为要素被判 infeasible 就删除或复用其编号。

## 七、提取自检声明填写说明

第八节的四项自查勾选是提取完整性审计（`templates/audit/extract_audit.md` 的 C1–C6）的起点材料，**不是走过场**：
- 「逐页扫描确认」与 C2（逐页覆盖扫描）对应；
- 「图表编号全登记」与 C1（图表编号连续性）对应；
- 「摘要结论数值可定位」与 C4（计数交叉）+ C6（幻觉引用抽查）对应；
- 「无法提取内容清单」如实列出，不得留空冒充"全部提取成功"——留空但实际有遗漏，会在 spec_audit 阶段被 codex 盲提取交叉验证（两阶段协议）与 auditor C 系列检查暴露，按 K7（幻觉引用）/遗漏类 finding 计入 critical/major。
